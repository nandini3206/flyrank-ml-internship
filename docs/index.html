FlyRank Content Refresh Priority Prediction using Machine Learning

Author: Nandini Bhatt

Abstract

This project develops a machine learning system for identifying website content that should be refreshed based on historical performance signals. Historical engagement, traffic, CTR, keyword and trend features are used to predict refresh priority. Multiple supervised learning models were evaluated and compared against a rule-based baseline. The Random Forest model achieved the best validation performance and was selected for generating the final refresh queue. The resulting recommendations are intended to support editorial decision-making rather than automate content publishing.

Introduction

Large content websites contain thousands of pages that cannot all be reviewed manually. This project investigates whether machine learning can prioritize pages that are most likely to benefit from content refresh using historical performance metrics. The goal is to help editorial teams focus their review efforts on high-impact pages while keeping humans in the decision loop.

Research Question

Can a machine learning model rank content pages by refresh priority using historical performance metrics?

Data

The project uses the FlyRank Content Refresh Dataset containing anonymized historical content performance metrics across 30,000 content items and 32 clients. The dataset contains engagement, traffic, CTR, keyword and trend-related features drawn from trailing 90-day performance windows. Client and content identifiers are pseudonymized, and no private URLs, queries, or client names are included anywhere in this project.

Rows with missing or invalid engagement history were excluded from training. Trend-derived columns (trend_direction, trend_pct) were excluded from the feature set entirely, since the target label is itself derived from these columns — including them would leak the label into the features.

Methodology

The target label, is_declining_label, identifies content items whose performance trend is declining. Missing values were handled during preprocessing, and leakage-related features (see Data section) were removed before training.

Several supervised machine learning models — Logistic Regression, Decision Tree, and Random Forest — were trained and evaluated using a client-holdout validation split (20% of clients held out entirely from training) to prevent client-level leakage between train and test sets. All models were compared against a transparent, rule-based baseline scorer on the same split and the same metrics. The model with the strongest validation performance (Precision@50) was selected to generate the final ranked refresh queue.

Random Forest improved Precision@50 from 0.240 (baseline) to 0.740 — a substantial gain in ranking quality among the top-50 flagged pages. It was selected as the final model based on this metric, and used to generate the ranked refresh queue with per-item confidence scores.

Limitations

  • The model predicts refresh priority, not guaranteed business impact — a "declining" label is not a guarantee that a refresh will improve performance.
  • Performance depends on historical data quality and may shift if user behavior or search algorithms change.
  • Editorial review is still required before updating content; recommendations are decision-support, not automation.

Ranked Recommendations

High-confidence pages should be reviewed first. Medium-confidence pages should be manually inspected. Low-confidence pages can be monitored for future review. The system is designed to assist editors rather than replace human judgment.

Reproducibility

Full code and analysis: GitHub repository. The complete implementation, notebooks, and scripts are available there — running work/notebooks/capstone.ipynb reproduces the preprocessing, model training, and refresh queue generation end to end.
